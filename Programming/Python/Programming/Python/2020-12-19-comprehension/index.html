컴프리헨션과 제네레이터

컴프리헨션

  • for 루프를 한줄로 표현할수 있는 문법
  • 코드량이 적어져 손가락 건강에 좋다
  • 컴프리헨션은 파이썬 인터프리터에 최적화 되어있어서 속도가 빠르다
  • 컴프리헨션으로 간단하게 리스트, 딕셔너리, 세트를 만들수 있다

List 컴프리헨션

보통의 리스트 루프

리스트 컴프리헨션으로 표현

이렇게 한줄이면 된다. foo(var)는 리스트에 append() 될 값. for var in origin_list는 기존의 for문과 동일하게 생각하면 된다.

딕셔너리 데이터를 이용해서 컴프리헨션을 만들수도 있다.

또한 컴프리헨션에 조건문을 걸 수 있다.

기존 컴프리헨션 문법은 동일하게 사용하면서 뒤쪽에 if 비교대상 비교연산자 비교대상 을 넣으면 된다.

정리하자면, 컴프리헨션 표현식은 아래의 코드로 이해할수 있다.

Dictionary 컴프리헨션

딕셔너리도 컴프리헨션으로 만들수 있다.

리스트 컴프리헨션과 그리 다르지 않다. 다른점이라면 []로 둘러쌓인 문법이 {}으로 변한것뿐. for k, v in origin_list.items()는 일반적인 딕셔너리의 루프문과 동일하다. values(), keys()도 사용 가능하다.

Set 컴프리헨션

세트도 컴프리헨션으로 만들 수 있다.

복잡한 컴프리헨션

헤드 퍼스트 파이썬의 예제를 가져와서 복잡한 컴프리헨션을 구현해본다.

세개의 컴프리헨션으로 나누어져 있다. 왼쪽부터 평가해가면 되는것 같다.

  1. 딕셔너리 컴프리헨션을 만들고 있다.
  2. dest라는 키값과 오른쪽의 리스트 컴프리헨션의 키 값을 가진다.
  3. flights 딕셔너리의 값이 dest라는 값고 같을때 해당 값의 키를 리스트로 만든다(딕셔너리 컴프리헨션의 값이 된다) .
  4. dest라는 값는 flights 딕셔너리의 값을 집합 컴프리헨션으로 표현하여 리스트 앞의 리스트 컴프리헨션과 딕셔너리 컴프리헨션에서 사용한다.

하지만… 파이썬 코딩의 기술 Better way 8에서는

  • 리스트 컴프리헨션에서 표현식을 두개 넘게 쓰지 말자
  • 두 개 이상의 컴프리헨션과 다중 if 조건이 섞이게되면 다른 사람이 이해하기가 힘들어진다.
  • 조건 두 개, 루프 두 개. 혹은 조건 한 개와 루프 한개 정도면 된다. 이것보다 복잡해지면 일반적인 if문과 for 문을 사용하고 아니면 아래에 설명할 제네레이너 함수를 이용하여 작성하는것이 좋다.

라고 한다.

컴프리헨션과 제네레이터

리스트 컴프리헨션과 비슷한 코드가 괄호로 감싸여 있다면 그 코드는 generator이다.

컴프리헨션의 단점

컴프리헨션은 다른 처리를 진행하기 전에 모든 데이터를 만든다.

lst = [x * 2 for x in another_list] 같은 코드에서 lst에 another_list의 요소들을 2씩 곱한 리스트를 한번에 만든다. lst가 만들어지는 동안은 다른 작업이 진행되지 않고 잠시 멈춘다.

작은 데이터를 가공하는 컴프리헨션이라면 문제가 없지만, 데이터의 양이 매우 클 경우 그동안 아무것도 진행되지 않기 때문에 프로그램에 큰 문제가 될 수 있다.

제네레이터

리스트 컴프리헨션의 대괄호를 괄호로 바꾸면 제네레이터가 된다. 위에서 말했듯이 컴프리헨션은 데이터가 모두 만들때까지 for 루프에 데이터를 제공하지 않고, generator는 데이터가 준비되는대로 데이터를 제공한다.

실제로 위 코드를 컴프리헨션, 제네레이터로 각각 실행해보면 차이를 느낄 수 있다. 리스트 제네레이터는 실행후 잠깐 멈췄다가 한 번에 세 줄이 출력된다.

제네레이터는 한 줄씩 출력된다.

데이터가 많아져 최적화에 문제가 생길 여지가 있다면 제네레이터를 사용하면 좋겠다.

yield

제네레이터는 함수로도 사용할 수 있다.

아래의 get_from_urls라는 함수가 이상하게 생겼다. tuple을 리턴하는 함수인데 리턴문은 없고 이상한 yield라는 키워드가 있다. 제네레이터는 데이터가 준비되는대로 즉시 제공한다고 했는데, get_from_urls 함수의 for 루프내에서 데이터를 전달해야 한다. 하지만 return 키워드를 사용하면 함수가 종료된다. 그럴땐 yield 키워드를 사용하면 된다. yield를 사용하면 함수가 반복자를 호출할 수 있는 제네레이터 함수로 변횐된다.

  1. for 루프에 포함된 제네레이터 함수 get_from_urls는 urls 튜플레어 첫 번째 url을 이용해 서버로 get 요청을 보낸다. 서버가 http응답을 반환하면 yield문이 실행된다.
  2. yield문은 첫번째 url에 제네레에터에서 생성된 튜플을 반환하고, get_from_urls 함수는 잠시 기다린다.
  3. yield가 전달한 데이터가 for문의 호출에 도착하면 for 루프가 실행되어 print()문이 실행된다.
  4. 호출 코드의 for 루프 반복하면서 다시 get_from_urls를 호출한다.
  5. get_from_urls는 다시 깨어나 1.으로 돌아간다.
