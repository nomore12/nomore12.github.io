TDD 프로세스

TDD 프로세스는 다음과 같이 진행된다.

  • 기능 테스트(functional tests)
  • 단위 테스트(unit tests)
  • 단위 테스트-코드 주기(unit test-code cycle)
  • 리팩토링(refactoring)

기능 테스트는 어플리케이션이 동작하는지 아닌지를 판단하기 위한 궁극의 수단이다. 반면 단위 테스트는 이 판단을 돕기 위한 툴이라 할 수 있다.

이런식으로 TDD를 표현한 것을 ‘이중 반복 TDD(double-loop TDD)’라고 부르는 경우도 있다. 이 주제에 관한 다른 관점의 기사도 있다.

클린코드를 위한 테스트 주도 개발 5장까지

지난 포스트부터 5장까지 정리.

기능테스트

기능: 페이지에서 작업 목록의 작성과 확인

사용자 스토리

  1. 에디스는 투두리스트 웹 어플이 나왔다는 소식을 듣고 확인하러 간다.
  2. 웹 페이지 타이틀과 헤더라 To-Do를 표시하고 있다.
  3. 에디스는 바로 작업을 추가하려고 한다.
  4. “공작깃털 사기”라고 텍스트 상자에 입력한다.
  5. 엔터키를 치면 페이지가 갱신되고, 작업목록에 “1. 공작깃털 사기”가 추가된다.
  6. “공작깃털을 이용해서 그물 만들기”라고 입력한다.
  7. 엔터키를 치면 페이지가 갱신되고, 작업목록에 “2. 공작깃털을 이용해서 그물 만들기”가 추가된다.

selenium을 이용해 사용자 스토리 기능 테스트를 구현한다.

  • unittest를 임포트하고 unittest.TestCase를 상속받은 클래스를 만든다.
  • setUp(self), tearDown(self)를 오버라이드하여 셀레니움 웹 드라이버 열기와 닫기 처리를 한다.
  • 클래스 밖에 if __name__ == "__main__"에서 uniitest.main(warnings="ignore")로 기능 테스트가 실행 될 수 있도록 한다.

단위 테스트

Django의 django.test에서 TestCase를 임포트한다. TestCase를 상속받은 클래스를 만들어 테스트를 진행 할 수 있다.

홈페이지 테스트와 모델 테스트 클래스 두개에 7개 메서드를 이용해 테스트를 진행했다.

  1. 홈페이지 테스트
    1. test_root_url_resolve_to_home_page_view(self)
    2. test_home_page_returns_correct_html(self)
    3. test_home_page_only_saves_items_when_necessary(self)
    4. test_home_page_can_save_a_POST_request(self)
    5. test_home_page_redirects_after_POST(self):
    6. test_home_page_displays_all_list_items(self)
  2. 아이템 모델 테스트
    1. test_saving_and_retrieving_items(self)

근데 뷰에 문제가 좀 있다.

request.POST는 딕셔너리인데, 없는 키의 값을 가져오려고 하면 빈 값을 가져온다. 책 중간에는 request.POST.get('item_text', '')으로 잘 사용하다가 나중에 위와같이 바꼈다. 빈 값을 가져오면 Item을 만들지 못하게 해야되는데 왜 저렇게 한건지 모르겠다. 여튼 저렇게 하면 빈 값도 생성되기 때문에 안된다.

csrf

test_home_page_returns_correct_html(self) 단위 테스트를 만들며 csrf 문제로 고생을 좀 했다.

csrf란 웹사이트 취약점 공격의 하나로, 사용자가 자신의 의지와는 무관하게 공격자가 의도한 행위를 특정 웹사이트에 요청하게 하는 공격을 말한다.

django는 템플릿에 {{ csrf_token }}만 입력하여 간단히 처리할수 있다. 하지만 웹사이트에서 보여주는 html과 템플릿을 비교하는 테스트에서 문제가 발생했다.

{{ csrf_token }}은 form의 <input type='hidden'>으로 난수를 생성하는데, csrf가 html에 포함되어 계속 assertEqual을 통과하지 못했다.

어떻게 검색하다보니 같은 문제를 겪고있는 동지들이 해결방법을 잘 알려주었다. 그냥 csrf를 html에서 빼면 되었고, 친절하게 csrf 제거 코드까지 만들어놨다.
