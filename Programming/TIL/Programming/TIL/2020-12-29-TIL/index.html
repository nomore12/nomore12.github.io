꼬리 슬래시와 정규표현식

염소책 119페이지 ~ 120페이지

정규표현식은 re_path()를 사용해야 함.

URL용 정규표현을 수정해서 캡쳐그룹(capture cgoup)(.+)을 추가한다. 캡쳐그룹은 / 뒤에 나오는 모든 문자에 일치된다. 취득한 텍스트는 인수 형태로 뷰에 전달된다.

123 페이지

AssertionError: 301 != 302

욕심많은 정규표현식의 문제.

Django는 영구적 리디렉션(301)에 대한 내부적인 이슈가 있어서 슬래시가 누락되는 경우 문제가 발생한다. 이 경우에는 /lists/1/add_item/이 list/(.+)/에 의해 매칭된다. (.+)가 1/add_item을 캡쳐하기 때문이다. 결국 Django는 우리가 마지막 꼬리 슬래시를 원한다는 ‘훌륭한’ 추측을 한다.

이것을 URL에서 숫자만 추출하도록 수정하면 된다.

zsh와 oh my zsh를 설치했고, 위 링크를 이용해서 터미널 테마를 변경함.

zsh를 이용해 유저정보 삭제

터미널창에 user@suser-desktop$ 처럼 유저 정보가 길게 표시되어서 몇 뎁스만 들어가도 커맨드를 한줄에 입력하기 힘들어져서 ~/.zshrc 파일을 수정하여 user-desktop 부분을 날려버렸다.

위의 코드를 ~/.zshrc파일에 추가하면 유저이름만 표시되어서 터미널을 더 깔끔하게 사용할 수 있다. 아, 수정후에 source ~/.zshrc 를 실행해야 한다. 그리고 bash 대신 zsh를 사용하면서 alias를 죄다 ~/.zshrc로 옮겨와야 했다.

내일의 뻘짓으론 터미널에 syntax highlighting을 적용해볼까 한다.

그리고 터미널이나 소스코드 글꼴은 마소에서 만든 Cascadia가 보기에 깔끔한 것 같다. Cascadia Code는 ligatures를 지원해서 != 같은걸 좀 더 알아보기 쉽게 만들어준다. Cascadia Mono는 ligatures가 없는버전, Cascadia (Code|Mono) PL은 파워라인 심볼이 포함되어 있어서 oh my zsh 터미널에서 폰트가 깨지지 않는다.
