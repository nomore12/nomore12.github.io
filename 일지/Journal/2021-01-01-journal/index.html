AWS

ASW 가입

그냥 가입하고, 멀티팩터 인증만 활성해놨다.

생활코딩 AWS 강좌. 여길 참고하면 가입 -> ec2 사용 -> 보안(otp) 까지 알 수 있다. 나도 이 재생목록을 먼저 보고 계정을 만들었다.

프리티어 사용이 가능한 t2 micro 우분투 인스턴스를 만들고 aws 콘솔을 이용해 접속했다. 웹에서 우분투의 cli를 실행할 수 있었다. 그리고 일단 nginx만 설치했다.

인스턴스 정보에 보니 ip 주소가 있어서 접속해보니 Nginx의 환영문구가 보인다. 하지만 인스턴스를 중지했다가 시작 할때 마다 ip가 변경된다.

찾아보니 탄력적 ip 할당(Elastic IP)를 이용하면 고정 아이피를 쓸 수 있다고 해서 그것도 적용해봤다.

Elastic Ip 할당.

서비스 -> EC2 -> 네트워크 및 보안 -> 탄력적 IP 메뉴로 들어간다.

오렌지색 탄력적 IP 주소 할당 버튼을 누르면

위와같은 화면이 나타난다. 중간과정을 찍어놨으면 좋았을텐데… 여튼 저렇게 할당할 수 있다.

그리고 다시 탄력적 IP 메뉴로 가면 위와같이 탄력적 IP 주소가 생성된 것이 보인다.

그리고 위 그림의 탄력적 ip 주소를 클릭한다.

위와같이 탄력적 IP를 인스턴스에 연결하는 설정하는 화면이 나온다. 인스턴스를 클릭하여 연결 하고자하는 인스턴스를 클릭 후 연결 버튼을 누르면 연결된다.

이제는 인스턴스를 중지했다가 다시 실행해도 IP주소가 바뀌지 않고 유지된다.

DOMAIN

가비아에서 구매한 도메인을 aws 인스턴스에 연결했다. aws의 Route 53 서비스로 이동한다. 대시보드 바로 아래에 호스팅 영역 메뉴로 이동한다. 바로 호스팅 영역 생성 버튼을 눌러준다.

그럼 위와같은 화면이 나타나고, 도메인 이름에 내가 구매한 도메인명을 넣는다. 당장은 인스턴스를 많이 쓸 것 같지 않아 설명, 태그는 대충 작성하고 호스팅 영역 생성 버튼을 누른다.

만들면서 스샷을 안찍어놓은걸 이렇게 두고두고 후회한다. 그리고 어딘가에서 www를 입력해야 하는 부분이 있었는데… 기억이 안난다. 가비아에서 넣어야 했던것 같기도 하고…

여튼 다시 호스팅영역에 들어가면 위 화면같은게 나오는데, 빨간색으로 표시한 부분이 아마 www를 넣어서 추가된 부분일거고, 파란색 부분이 값/트래픽 라우팅 대상이다. 이제 값/트래픽 라우팅 대상을 가비아의 네입서버에 설정해줘야 한다.

가비아에 접속해서 My가비아에 들어가면 도메인 정보 변경이라는 메뉴가 있다. 도메인명을 클릭해서 도메인 정보를 수정해준다. 아마 안전잠금 이라는게 있어서 인증 후 수정해야 한다. 그리고 위의 파란부분에 있던 네 개의 조소를 네임서버 네 군데에 입력하면 끝.

음… 너무 날림으로 작성한 것 같다. 그리고 이게 과연 TIL이 맞는지도 모르겠다. 그냥 aws 포스팅감 아닌가?
