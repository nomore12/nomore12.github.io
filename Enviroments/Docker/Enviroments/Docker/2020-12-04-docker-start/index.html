도커 이미지와 컨테이너

개념 컨테이너
도커 이미지 도커 컨테이너를 구성하는 파일 시스템과 실행할 애플리케이션 설정을 하나로 합친 것. 컨테이너를 생성하는 템플릿 역할을 한다.
도커 컨테이너 도커 이미지를 기반으로 생성되는 파일 시스템과 애플리케이션이 실행되고 있는 프로세스

도커 이미지 생성하기

컨테이너를 만들기 전에 먼저 도커 이미지를 받아온다.

안된다…

아무래도 터미널에서 로그인을 하라는것 같다. 찾아보니 docker login 명령으로 docker.com에 로그인 할 수 있는것 같다.

로그인은 되었는데 무언가 문제가 있다. 보안 문제인 것 같다. 이걸 해결하려다가 너무 많은 시간을 쓸 것 같아 일단 넘어간다. 도커를 익히는 것이 먼저다.

이미지를 잘 받아왔다. sudo docker image pull imagename/tag 형식으로 이미지를 받아올 수 있다. 위에선 tag를 입력하지 않았지만, 알아서 latest 이미지를 받아온것을 알 수 있다.

도커 허브의 ubuntu페이지에 가보면 위 그림과 같은 태그 정보가 있다. latest는 이미지가 업데이트되면 버전이 달라질 수 있으니 명시적으로 tag를 입력해주는 것이 좋을것 같다.

도커 컨테이너 생성하기

받아온 도커 이미지로 부터 컨테이너를 생성하기 전에 받아온 이미지를 확인한다.

무언가 잘 못 되었다.

/var/run/docker.sock의 권한 문제인 것 같다.

방금 받아온 ubunut/latest 이미지를 확인할 수 있다.
이제 컨테이너를 만들어보겠다.

컨테이너가 만들어졌다. -i -t같은 옵션이 사용되었는데, 이건 나중에 알아보자. --name myubuntu는 컨테이너의 이름을 지어주는 옵션이다.

위에 출력된 16진수 해시값은 컨테이너 고유의 ID이다. 컨테이너가 잘 만들어졌는지 확인해보자.

컨테이너를 create 할 때 출력된 해시값 앞의 12자리와 이미지 이름, 커맨드, 생성 시간, 상태, 포트, 이름 항목이 표시된다. 활성화된 컨테이너 목록을 보려면 docker ps 만으로 충분하지만, docker create로 생성한 컨테이너는 생성만 되고 실행되진 않는다. 실행하려면 docker start 명령어를 입력해야 한다.

docker ps 명령어만으로 컨테이너가 발견되었다. STATUS도 Up 2 senconds라고 표시되어 myubuntu 컨테이너가 2초전에 실행된 것을 알 수 있다.

이제 myubuntu 컨테이너 안으로 들어가보자.

컨테이너 bash에서 작업을 해야할 때도 있을것 같은데, exit로 나오면 컨테이너가 죽어버린다. 이럼 안될것 같아서 방법을 찾아보았다. 컨테이너를 정지하지 않고 빠져나오는 방법으로는 Ctrl + P, Q 를 입력하면 된다고 한다. Ctrl + P 를 먼저 누르고, P만 뗀 후 Q를 누르면 된다.

도커 컨테이너 삭제하기

myubuntu 컨테이너 이름도 맘에 안들고, 왠지 꼴도보기 싫어서 삭제하고 싶어졌다. 컨테이너를 삭제하려면 먼저 컨테이너를 정지해야 한다.

컨테이너 목록에 아무것도 없는것을 확인했다.
