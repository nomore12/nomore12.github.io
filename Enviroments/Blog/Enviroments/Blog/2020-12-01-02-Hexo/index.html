Hexo를 이용하여 github.io 블로그 만들기.

Hexo

마크다운 문서를 이용한 블로그를 만들고 싶었다. 원래 계획은 Django를 이용하여 처음부터 끝까지 블로그를 만드는 것이었지만… 다른거 공부할 것도 많은데 그 고생을 해야하나 싶었다. 그래서 처음에 찾은건 jekyll. 하지만 테마가 마음에 드는 것이 없었다. 그래서 찾은게 Hexo의 icarus 테마. 그리고 jekyll은 루비기반이라(굳이 루비를 알아야 하진 않겠지만) 맘에 안들었는데, Hexo는 Node.js기반이라 맘에 들었다.

Hexo 설치

Hexo는 node.js 기반이라 npm 명령어를 통해 간단하게 설치할 수 있다. 헥소 홈페이지를 참고해본다. 그리고 이분의 도움도 많이 받았다.

초기화가 완료되면 아래와 같은 구조로 셋팅된다.

  1. 테마를 클론
  2. _config.yml파일에서 테마 수정.

스캐폴드

포스트 또는 페이지의 구조를 설정할 때 사용한다. scaffolds 폴더의 draft.md, page.md, post.md 파일을 수정하면 된다.

기본값은 위와같이 되어있다. 기본값에서 태그를 추가하려면 포스팅할 md파일에서

이렇게 추가해야 한다. tags: [ ] 로 입력하면 줄바꿈하지 않고 배열처럼 입력 가능하다. 그리고 카테고리와 목차를 추가했다.

카테고리와 toc를 위와같이 추가하고 hexo new postname으로 포스트를 추가하면 추가된 md파일이 아래와 같이 입력되어 있다.

여기에 카테고리를 추가해준다.

카테고리 위젯에 Enviroments 대분류가 생성되었고, Blog 소분류가 생성되었다. 줄바꿈하여 카테고리를 나열하면 대분류 -> 소분류로 생성된다.

toc: true는 카탈로그 위젯에 H1등의 헤더들을 목차로 보여준다. 헤더는 H1 ~ H6까지 있지만 이카루스 테마의 _config.icarus.yml파일을 수정하여 depth를 지정할 수 있다.

  • position은 포스트의 오른쪽 공간에 위젯을 배치하는 옵션이다.
  • index는 목차에 번호를 붙여준다.
  • collapsed는 하위 헤더의 위치에 있지 않다면 상위 헤더 아래로 접어주는 옵션이다.
  • depth는 목차에 헤더의 깊이를 얼마나 보여줄것인지 정해주는 옵션이다. depth가 3이니 H3까지만 보여준다.
    VSCode의 기본 설정으로 스캐폴드를 수정하니 title: { { title } }로 바꿔버렸다. 별 문제 없거니 하고 포스트를 생성했더니 뭔가 잘못되고 있었다. hexo new text로 새 포스트를 생성해보았다.

title을 입력하는 부분이 맛이 갔다. JSON 스타일로 변했다. 저대로 포스트를 내보내면 '[object Object]'가 타이들에 그래도 입력된다. 그래서 VSCode에 Markdown All in One이라는 확장을 설치했다. 설치한 후에 {{ title }}로 수정하고 저장하니 정상적으로 저장되었다.

VSCode의 Markdownlint

VSCode 확장 프로그램인 Markdownlint를 설치하면 마크다운 문법이 어디가 잘못되었는지 알려주고 자동으로 수정해주기도 한다.

위 그림처럼 오류가 잔뜩 잡힌다. 그래서 문법 오류를 잡아주는 단축키 Ctr + . 를 눌러주면 아래와 같은 화면이 나온다.

위와 같은 Ctr + . 를 누른 오류 아랫쪽에 4개의 메뉴가 나타난다.

첫 번째 메뉴를 선택하면 해당 오류를 자동으로 해결해준다.

하지만 위 그림처럼 자동으로 수정할 수 없는 경우도 있다.

  1. 첫 번째 메뉴는 markdownlint의 규칙 문서 페이지로 이동한다. 해당 항목을 보고 수정하면 된다.
  2. 두 번째 메뉴는 설정창을 열어준다. settings.json을 이용해 markdownlint의 오류식별자(MD025 등)을 수정할 수 있다.
  3. 세 번째 메뉴는 markdownlint의 설정 메뉴얼 페이지로 이동한다.

hexo에서 heading 태그의 오류를 잡는게 조금 골치아팠다. 상위 레벨의 # 태그가 있으면 하위 레벨의 ## 태그가 반드시 있어야 한다는 것이다. 근데, # 태그 아래에 아무리 ## 태그를 넣어도 오류가 사라지지 않았다. 나중에 알게된 것이지만 hexo는 문서의 title이 # 태그로 작성되어 있어서 해당 문서의 heading 태그를 적용하고 싶으면 ## 태그부터 시작해야 한다.
