HEXO 백업

백업링크

위의 링크를 참고하여 백업.

  1. theme를 저장할 리파지토리를 만든다. theme이름과 동일하게 만드는게 오류가 안나고 좋더라.
  2. themes/icarus의 리모트를 1.에서 만든 리파지토리로 변경하고 올린다.
  3. hexo 프로젝트 폴더에 git 저장소를 만들고, public, .deploy_git, node_modules를 ignore한다.
  4. 2.에서 제대로 백업이 되었는지 확인한 후 themes 폴더의 icarus 폴더를 지운다.
  1. 그리고 themes 폴더에서 git submodule add https://github.com/repository_url.git을 입력해 서브모듈로 추가한다.
    • 새로운 테마를 설치할 때도 clone보다 submodule로 설치하는게 더 좋다고 한다.
  2. hexo 프로젝트 폴더의 변경사항을 모두 커밋해서 새 리파지토리로 올린다.
  3. 리파지토에서 잘 백업이 되어있는걸 확인할 수 있다.

깃헙의 hexo 프로젝트 리파지토리 themes 폴더에 가보면 icarus @ 커밋아이디 폴더가 생성되어있다. 폴더를 클릭하면 나의 icarus 리파지토리로 바로 이동한다.

.gitsubmodule파일은 이렇게 생겼다.

path=themes/icarus, url=icarus테마 리파지토리 주소로 프로젝트 루트가 서브모듈의 리파지토리, 커밋을 찾아가는듯 하다.

그리고 서브모듈을 수정했으면 서브모듈을 먼저 커밋하고 푸쉬한다음에, 루트를 커밋하고 푸쉬해야 한다. 서브모듈은 커밋만 하고 루트에서 커밋 푸쉬하면 다 되겠지 했는데… 안된다. 레파지토리의 서브모듈의 head는 서브모듈의 최신 커밋을 가리키고 있지만 클릭해보면 해당 커밋은 푸쉬되지 않았기때문에 없는 페이지라고 나온다. 서브모듈과 루트를 모두 수정했다면, 서브모듈 커밋 -> 서브모듈 푸쉬 -> 루트 커밋 -> 루트 푸쉬 순서로 해야한다.

그냥 따라만하니 예상하지 못했던 오류가 자꾸 발생한다. 근데 그렇다고 전부 다 공부한다음에 하려면 언제 시작할 수 있나 싶기도 하고.
