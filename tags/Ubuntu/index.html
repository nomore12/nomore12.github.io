우분투 멀티부팅 환경 만들기

0. 개발환경 셋팅

지난 6월에 찾아온 TFCC 덕분에 맥북을 못 쓰게 되었다. 6 ~ 10월 까지는 재활 말고는 할 수 있는게 없어 매우 좌절스러웠다. 그래도 이제 각종 인체공학 장비를 이용하면 어느정도 개발이 가능해져서 개발환경부터 셋팅하려고 한다.

1. 우분투 설치

1. 부팅 USB만들기

우분투 ios파일을 다운받고, RUFUS를 이용하여 부팅 USB를 만들었다.

bios에서 usb부팅 설정 후 우분투를 설치했다. 잘 설치된 듯 했으나 무한 로그인 루프에 빠졌다. ctrl + alt + f1을 이용해 TTY를 열어 해결하는 방법도 있다고 하는데… 왜 난 ctrl + alt + f1을 눌러도 TTY 화면이 나오질 않는걸까…

그래서 그냥 재설치 했다.

잘 되네.

2. 필수 프로그램 설치

1. VScode 설치 및 환경 동기화

  • VScode의 Settings Sync 확장 프로그램을 이용해 윈도우 vscode의 셋팅을 우분투로 옮겨왔음. 이곳을 참고해서 셋팅함.
  • 그리고 기존 셋팅에서 폰트를 마이크로 소프트에서 만든 Cascadia Code를 쓰고 있었음. /usr/share/fonts/opentypes 폴더에 옮겨넣어 폰트 설정 완료.
  • /usr/share/fonts/opentype, /usr/share/fonts/truetype 폴더에 otf, ttf 폰트가 포함된 폴더를 옮기면 된다.

1. 프로그램 설치

크롬과 Apache Open Office를 설치했다.

  • 근데 예전에 Ubuntu Software에서 VSCode를 다운받으면 한글 입력이 제대로 되지 않았던 기억이 있어서 .deb파일로 다운받아 설치했다. 우분투 소프트웨어는 뭔가 믿음직스럽지 못하다.

  • Apache Open Office 설치(쓸 일이 있을까 싶지만 일단 설치.)
    아파치 오픈오피스 설치파인 다운로드 후 기존에 설치되어 있는 리브레 오피스 삭제

  • 다운받은 오픈 오피스 압축 해제하고 해제한 폴더에서 패키지 일괄 설치

3. 한글입력 삽질

한글 입력을 제대로 하기 위해 수많은 삽질을 했다. 처음엔 아예 한글 입력이 되지 않다가, shift + space로만 한영 전환이 되다가, 다른 모든 프로그램에선 오른쪽 alt로 한영전환이 되는데 크롬에선 안된다던가…

여튼 방법을 찾긴 했다. uim을 일단 설치한다. $ sudo apt install uim

한글 오타를 방지해주는 uim-byeoru를 설치한다. $ sudo apt install uim-byeoru

언어 지원(language support) 프로그램을 실행하면 아래와 같은 창이 뜬다.

한국어가 추가되었는지 확인하고 없다면, 언어 설치/제거를 이용해 추가해준다. 그리고 키보드 입력기를 uim으로 변경해준다.

uim-pre-gtx라는 프로그램을 찾아서 실행하면 아래와 같은 화면이 나옴다. 디폴트 입력기를 벼루로 바꿔준다.

이제 한글 입력 키를 바꿔줘야 한다. 한영 전환 디폴트 값은 Shift + Space다. 윈도우 사용자도 그렇고, 맥 사용자도 Shift + Space는 적응하기 진짜 힘들기 때문에 아래의 벼루 키 설정에서 한글모두, 영문모드를 오른쪽 Alt키로 변경해야 한다.

저리 설정하면 오른쪽 Alt키가 한글키로 잘 작동한다고도 하는데 나는 그렇지 않았다. 그래서 xmodmap이라는 프로그램으로 키 맵핑을 바꿔줬다.

터미널에서 $ xmodmap -pke > ~/xmodmap_original 명령어로 기존 키보드 레이아웃을 백업해놓는다.

4. 상단 바, 작업 표시줄

gnome-tweak-tool을 이용해 상단 바를 조금 커스터마이징 했다. 나는 거의 의미없는 수준… 이었지만, 누군가는 잘 활용하겠지. 아, 핫 코너는 조금 좋았다.

작업 표시줄(pannel)이 계속 주 모니터에서만 나오고 오른쪽에 붙어있는게 너무 마음에 안들었다. 그래서 검색해보니 여기에 환경설정에서 설정 가능했다. 모양 메뉴의 표시라고 쓰여있어서 계속 찾았는데 다른 설정이라고 생각하고 그냥 지나친 듯 했다.

5. 윈도우, 우분투 부팅 순서

우분투를 설치하면 부팅 우선순위가 우분투다. 개발을 주로 할거면 그것도 상관없긴 한데, 돌겜도 해야되고 윈도우에 이것저것 다른것도 많아서 그냥 윈도우를 우선순위로 하고 싶었다. 그리고 우분투가 지맘대로 부팅 우선순위를 바꾼것도 맘에 안들었다. 지가 뭔데 맘대로 바꾸고 있어.
우분투는 GRUB 메뉴라는걸 이용해 부팅 순서를 선택한다는 것 같다.

찾아보니 /etc/default/grup 파일을 수정하면 GRUB 부팅순서를 쉽게 변경 할 수 있었다.

그리고 sudo update-grub명령을 입력하여 변경된 설정을 적용한다.

6. /home/ 내부 폴더명 영문으로 바꾸기.

한글 우분투를 설치하면 /home/ 내부의 폴더명이 전부 한글이다. 터미널에서 $ cd 문서 이러고 있으면 개빡친다. 그래서 일단 폴더명을 영문으로 바꿔봤다. 그리고 웹에서 파일을 다운받으니 파일이 다시 생성된 다운로드^^ 폴더에 다운되었다. 그래서 검색해보니 이곳에서 해답을 찾을 수 있었다.

결론적으로 ~/.config/user-dirs.dirs 파일을 수정하면 된다.

7. 특정 프로그램 설치 시 CD-ROM 요구 수정

어떤 프로그램(스팀)을 설치하다보면 터미널에서 cd-rom을 넣으라고 할 때가 있다. 아니 무슨 90년대도 아니고 cd-rom을 찾는지 모르겠다.
여튼, $ sudo nano /etc/apt/sources.list를 수정하면 된다.

8. 기본 키 모음

부팅 할 떄마다 기본 키 모음이 잠겨있다고 계속 비밀번호를 입력하라고 한다. 계속 검색해서 이것저것 해보았지만 안됨. 이건 빡쳐서 안되겠다. 나중에 함.

9. Node.js 설치

패키지 매니저를 이용해 node.js를 설치하였다.

setup_14.x에 버전 정보를 입력하면 해당 버전으로 설치가 된다. 14.x로 설치하니 14버전의 최신버전인 14.15.0 LTS 버전으로 설치되었다.
