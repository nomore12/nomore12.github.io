Selenium

클린 코드를 위한 테스트 주도 개발에서 셀레니움을 이용해 테스트 했던게 기억나 도커파일에 셀레니움 설치 코드를 추가했다. 가벼운 파이썬 이미지에선 apt-get이 없었고, apt-get이 있는데선 크롬 패키지가 없었다. 여차저차해서 설치했다.

근데 DevToolsActivePort를 찾을수가 없단다. 찾아보니 도커 이미지에서 셀레니움을 돌리면 크롬 브라우저가 떠야되는데 이미지에 gui 데스크탑이 없으니 안되는거였다. 아래와 같은 옵션만 추가하면 된다.

그리고 나서 셀레니움으로 url 유닛테스트 까지 성공했다.

그런데…

Flask 클래스의 어트리뷰트로 data가 있는데… 이게 html 텍스트 였나보다… 플라스크 공식 문서를 제대로 봐야겠다…

Flask-restful

Django에 Django restframework가 있는것 처럼 Flask에도 Flask-restful이 있다.

이렇게 간단하게 /user의 post api를 만들었다.

간단한건 좋은데, 저 api.add_resource(ResisterUser, '/user')가 너무 거슬린다. api가 늘어날수록 저 코드가 하나씩 늘게되는데 그건 하지 말아야 할 것 같고, api 객체를 다른 파일에서 임포트하고 그 파일에서 add_resource를 한 다음에 다시 app.py로 가져오면… Circular Import가 되서 안될것 같다. 찾아보니 런타임에서 임포트 하는 방법이 있는것 같긴 한데, 그것도 방법이 아닐것 같다.

아직 플라스크도 제대로 모르지만 Flask-restful 공식문서의 Intermediate Usage를 찾아보니 before_request(), after_request()를 이용할 수 도 있을것 같다. before_request()에서 request url을 알 수 있으니 그때그때 add_resource를 한다던가… 일단, 그건 나중에 더 알아보는 것을로 하고, 다시 코드를 보다보니… 아니 validation은 어떻게하지?;

찾아보니 validation은 marshmallow라는 라이브러리를 사용하면 되는것 같다. 그나마 다행이다.

그리고 생각해보니… 봉봉 유저가 보는 페이지는 그리 어렵지 않을것 같은데, 관리자 페이지를 생각하니 눈앞이 깜깜해졌다. 가볍게 시작하려고 했는데 생각보다 할게 너무 많은것 같다.

그냥 Django로 갈아타야되나 심각하게 고민이 된다.
